In the main [GitHub Repository](https://github.com/GitBrincie212/ChronoGrapher) under the ``docs`` folder lies the website,
there are many folders and files but a few are important:
- ``content/docs`` The place where the actual documentation resides in.
- ``public`` This is where images go, such as diagrams, animations... etc.
- ``src/components`` Includes the React components (and ones that are meant for MDX) reside in.
- ``src/assets`` Unlike public, assets is commonly used for anything in the landing page itself.
- ``src/app/docs/[[...slug]]`` Contains the code that handles displaying a documentation page (like this one).
- ``src/app/(home)`` Has the code for the landing page.

The top 4-5 bullets are the most important of all. As these folders will be something you interact almost daily,
we will be mainly focusing on the top 2 which are ``content/docs`` and ``public`` folders.

The ``content/docs`` includes sub-folders and 2 filetypes commonly encountered, these are ``.mdx`` files and ``meta.json``
files, both serve different roles with the most important being ``.mdx``.

The MDX file is similar to any typical [Markdown](https://www.markdownguide.org) file, however it has the power to use
React components and define a title and a description for its page.

The format of an MDX file is:
```mdx title="my_document.mdx"
---
title: My Title XYZ
description: Lorem ipsum dollar - some dummy description
---

<...>
```
<Callout type={"idea"} title={"Marking Page / Chapter As Optional"}>
    Some chapters are meant to be optional or "extra-content", for this reason the ``optional`` tag exists,
    which inserts a badge, to use it simply type ``optional: true`` in-between the ``---`` block.
</Callout>

It should be mentioned that ``My Title XYZ`` is the title and ``Lorem ipsum dollar - some dummy description`` is the
description while also the ``<...>`` is simply your Markdown / MDX content.

There is a special filename called ``index.mdx``, this is entry root file which displays when users go in the
specific folder, in our case its [Introduction & Prerequisites](/docs/contributing).

For the ``meta.json`` files, these are used to organize explicitly, they indicate the folder as a collapsible with the
same name as the folder, they have the format.
```json title="meta.json"
{
  "title": "My Title ABC",
  "pages": [
    "document1",
    "document2",
    "document3"
    // <...>
  ],
  "icon": "MyIcon",
  "defaultOpen": true
}
```
Where the title is ``My Title ABC``, the icon which is optional is ``MyIcon`` and we specify that its default open via
``defaultOpen`` property, and the documents it includes belong in the folder and can be any number of them.

<Callout type={"info"} title={"For Adding Icons"}>
    Though its uncommon practice to add icons in the collapsibles, you may generally need to add for something, for this reason
    check the [Loader API From Fumadocs](https://www.fumadocs.dev/docs/headless/source-api#icons).
</Callout>
